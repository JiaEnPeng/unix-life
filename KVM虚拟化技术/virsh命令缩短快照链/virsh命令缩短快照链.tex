\section{virsh命令缩短快照链}
	使用virsh命令动态创建快照的方法在《动态创建快照的两种方法》中已经提过了，在这篇文档中介绍的是，
	当镜像链为4时，如何将镜像链的长度变为3。\par

	假设我们现在的镜像链为base <- overlay1 <- overlay2 <- overlay3：
	\begin{itemize}
		\item[1.] 首先将overlay2融入overlay1中：
		\begin{lstlisting}
	# overlay是虚拟机的名字
	sudo virsh blockcommit --domain overlay hda --base overlay1.qcow2 --top overlay2.qcow2 --wait --verbose
		\end{lstlisting}

		\item[2.] 清除原先快照2的元数据：
		\begin{lstlisting}
	sudo virsh snapshot-delete overlay snap2 --metadata
		\end{lstlisting}

		\item[3.] 这两个步骤以后，可以看到overlay3.qcow2的镜像链如下：
		\fic{1.png}
	\end{itemize}

	需要知道的是，使用virsh命令动态创建快照时，不会影响虚拟机的运行。使用virsh命令缩短镜像链时，也不会影响虚拟机的运行。\par
