\section{使用qemu-img工具}
    使用qemu-img工具动态创建快照的步骤如下：
    \begin{itemize}
        \item[1.] 假设一台虚拟机正在运行，它的磁盘镜像为devstack.qcow2，使用如下命令将它更名为devstack.qcow2.base：
        \begin{lstlisting}
    mv devstack.qcow2 devstack.qcow2.base
        \end{lstlisting}

        \item[2.] 根据虚拟机的磁盘镜像创建一块增量镜像，这块增量镜像的名称为devstack.qcow2：
        \begin{lstlisting}
    qemu-img create -f qcow2 -b devstack.qcow2.base devstack.qcow2
        \end{lstlisting}
    \end{itemize}

    此时，如果不重启虚拟机，那么虚拟机上数据的改动将同时记录到devstack.qcow2.base和devstack.qcow2中。\par

    当重启虚拟机后，虚拟机上数据的改动只会记录到devstack.qcow2中，也就是新创建的增量文件中。

\section{使用virsh命令行}
    使用virsh命令行动态创建快照的步骤如下：
    \begin{itemize}
        \item[1.] 假设一台虚拟机正在运行，它的磁盘镜像为devstack.qcow2，使用如下命令就可以动态创建快照：
        \begin{lstlisting}
    # devstack是虚拟机的名字
    # snap是快照名
    # snap-desc是快照描述
    # file=/home/pengsida/kvm/openstack/overlay.qcow2用于指定新的增量文件
    virsh snapshot-create-as devstack snap snap-desc --disk-only --diskspec hda,snapshot=external,file=/home/pengsida/kvm/openstack/overlay.qcow2 --atomic
        \end{lstlisting}
    \end{itemize}

    这个方法可以真正地实现动态创建快照，也就是执行上述命令后，虚拟机数据的改动将记录到overlay.qcow2文件中，不会记录到devstack.qcow2文件中。\par

    不过值得一提的是，openstack中都是使用“qemu-img”工具，没见过使用“virsh”命令行。
