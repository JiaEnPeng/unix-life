\section{第一个汇编程序}
	首先看一段汇编语言源程序：
	\begin{lstlisting}
	assum cs:codesg

	codesg segment

	start:
		mov ax, 0123H
		mov bx, 0456H
		add ax, bx

		mov ax, 4c00H
		int 21H

	codesg ends

	end start
	\end{lstlisting}

\subsection{伪指令}
\subsubsection{segment和ends}
	segment和ends用于定义一个段，segment说明一个段的开始，ends说明一个段的结束，使用格式如下：
	\begin{lstlisting}
	段名 segment
	; ...
	段名 ends
	\end{lstlisting}

	一个汇编程序由多个段组成，有代码段、数据段、栈段。一个汇编程序至少要有一个代码段，用于存放指令。

\subsubsection{end}
	end是一个汇编程序的结束标记，当编译器遇到伪指令end时，就会结束对源程序的编译。\par

	end除了通知编译器程序结束外，还可以通知编译器程序的入口在什么地方。
	比如第一个汇编程序中end指令就指明了程序的入口在标号start处。

\subsubsection{assume}
	assume会把有特定用途的段和相关的段寄存器关联起来，从而让编译程序将段寄存器和某一个具体的段相关联，如下例所示：
	\begin{lstlisting}
	; codesg是段名，cs是段寄存器
	assume cs:codesg
	\end{lstlisting}

\subsection{标号}
	除了伪指令、指令之外的符号就是标号，比如“start”。
	一个段名也是一个标号，比如“codesg”，一个标号指代了一个地址，段名代表着一个段的段地址。
