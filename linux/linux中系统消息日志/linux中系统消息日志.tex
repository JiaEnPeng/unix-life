\section{linux中系统消息日志}
\subsection{查看linux系统消息日志}
	有两种方式：
	\begin{itemize}
		\item[1.] dmesg查看。
		\item[2.] 查看/var/log下的文件。
	\end{itemize}

\subsection{dmesg的工作原理}
	dmesg是从kernel 的ring buffer(环缓冲区)中读取信息的。\par

\subsubsection{什么是ring buffer}
	在LINUX中,所有的系统信息(包内核信息)都会传送到ring buffer中.
	而内核产生的信息由printk()打印出来。
	系统启动时所看到的信息都是由该函数打印到屏幕中。 
	printk（）打出的信息往往以<0><2>...这的数字表明消息的重要级别。
	高于一定的优先级别会打印到屏幕上， 否则只会保留在系统的缓冲区中(ring buffer)。

\subsection{/var/log下文件的产生}
	 LINUX系统中/etc/init.d/sysklogd会启动2个守护进程:Klogd&&Syslogd。\par

	 klogd是负责读取内核信息的,有2种方式:1.syslog()系统调用；2.直接的对/proc/kmsg进行读取。\par

	 Klogd的输出结果会传送给syslogd进行处理,
	 syslogd会根据/etc/syslog.conf的配置把log信息输出到/var/log/下的不同文件中.\par


	 如果klogd没有运行，内核消息就不会传递到用户空间，这种情况下只能查看/proc/kmsg文件。
